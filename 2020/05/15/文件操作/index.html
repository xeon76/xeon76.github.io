open简介

open(file,mode,buffering,encoding)函数
mode: r:read读 w:write写 处理纯文本文件
rb:read binary wb：write binary 处理纯文本，图像，音乐，电影
常用模式
‘r’ open for reading (default)
‘w’ open for writing, truncating the file first
‘x’ create a new file and open it for writing
‘a’ open for writing, appending to the end of the file if it exists
‘b’ binary mode
‘t’ text mode (default)
‘+’ open a disk file for updating (reading and writing)
‘U’ universal newline mode (deprecated)
默认rt

文件的写操作

stream = open(r”C:\Users\22107\Desktop\文件操作数据\aa.txt”,”w”)
mode是”w”表示是写操作
方法:
write(内容) 每次都会将原来的内容清空，然后写入当前内容，这里的每次是指创建一个流到关闭一个流，在这个期间连续写入是不会清除的
writelines(Iterable) 没有换行效果,如果要实现换行效果，自己加\n
mode是”a”时，不会清空原来内容，在原来的基础上进行添加

如何将上面那个例子中的图片保存到当前文件夹下

绝对路径：c:\p1\girl.jpg
os.path.isabs(a) #判断a是否是绝对路径
相对路径：相对路径是有参照点的，这个参照点就是这个py文件所在的位置
—image
  —girl.jpg
—当前py文件.py
那么这个girl.jpg的相对地址是”image/girl.jpg”

如果
—image
  —girl.jpg
—code
  —当前py文件.py
那么这个girl.jpg的相对地址是”../image/girl.jpg” ..是当前文件的上一级的意思

os.path.abspath(a) a是一个相对路径，这个功能是通过相对路径获取绝对路径
os.path.abspath(file) 获取当前py文件绝对路径，这边file就是代表当前文件 c:\p1\1.py
os.getcwd() 不需要输入参数，当前py文件所在文件夹路径（绝对路径） c:\p1
os.getcwd()类似于os.path.dirname(file)
要了解上面两个的区别
os.path.split(path) 得到一个元组，将文件与文件所在的目录切割开来了
比如os.path.split(“c:\p1\girl.jpg”) => (“c:\p1”,”girl.jpg”)
os.path.splittext(path) 分割文件与扩展名
比如os.path.splittext(“c:\p1\girl.jpg”) =>(“c:\p1\girl”,”.jpg”)
os.path.getsize(path) 获得文件的大小，单位是字节
